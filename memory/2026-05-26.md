# 2026-05-26 开发日志

## 上午 - 移动端 UI 批量优化

### 鸟详情页（bird.ejs）
- **CSS 换行符故障修复**：Node.js `\n` 转义写成了文字 `\n`，浏览器无视整个媒体查询。尝试二进制修复导致 UTF-8 编码损坏 + 500 错误。最终用 `Node.js .replace(/\n/g, '\n')` 修复（`\n` 文本→真正 LF 0x0A）
- **手机端布局**：`flex-direction: column` 照片在上，`display:contents + order` 重排：名字+环号→摘要→按钮→照片
- **环号右上角**：`margin-left: auto; color: #b8a898; font-size: 0.8rem`
- **描述区摘要化**：`公 · 1.1岁 · 存活` + `绿花桃 · 已配对 · 1号繁殖间` 两条摘要行，meta-grid 手机端隐藏
- **按钮统一**：`.btn-edit` / `.btn-medical` / `.btn-success` 共用 `min-height` / `border-radius` / `padding`
- **病历页**：grid 单列、header 折行、编辑/痊愈按钮→右上角 absolute、时间轴 scroll-snap + 渐隐遮罩、添加条目全宽按钮、datetime-local 精确到分钟
- **首页卡片精简**：去掉 species 标签、年龄 `1.1岁` / `3个月` / `401天`、统一 pill 标签、性别图标 32→26px

### 导航栏（topbar.ejs）
- **汉堡菜单→侧滑抽屉**：checkbox hack → `position: fixed` 抽屉 82vw/320px + `rgba(0,0,0,0.45) + backdrop-filter: blur(4px)` 遮罩
- **分组标签**：主功能 / 系统 / 账户，`nav-group-label` 小字分组
- **header 紧凑**：`padding 20→14px`，字体 `17→15px`
- **汉堡按钮轻量化**：`36→32px`

### 编辑模式优化
- **EJS 损坏修复**：`fix-*.js` 脚本的 `\r\n` vs `\n` 差异导致 EJS 模板被替换成破损内容，裸露 `<%` 标签显示为文本。回退到 commit 5391755 恢复
- **表单分组**：`form-group > h4` 基础/状态/其他信息三段
- **父母选择→卡片**：`parent-chip` 整块可点击（高度44px、圆角12px、outline风格）
- **按钮降权**：编辑/病历 `min-height 44→36px; border:none; transparent`
- **编辑模式**：`body.editing` 隐藏 overview + 图片 + 病历按钮
- **取消按钮**：新增 outline 风格取消按钮
- **删除弱化**：outline 风格放到底部
- **体重 API**：`/bird/api/:id/weights`（复数 s）

## 坑记录
1. CSS `\n` 文字 vs 换行符 — 浏览器无视媒体查询
2. 二进制替换移字节位 → 炸 UTF-8 编码
3. Node.js 正则 `/\n/g` vs `/\\n/g` 匹配差异
4. EJS 必须在文本层面（字符串 replace）操作，不能用二进制字节替换
5. 编写修改 EJS 模板的脚本要注意 `\r\n` vs `\n`
6. `edit` 工具可靠但 Node.js inline `-e` 在 PowerShell 下引号逃逸是灾难

### drift 踩坑
- `insert()` 用裸值，`write()` 才用 `Value<>` 包装（API 不一致）
- 生成的数据类名：`Species` → `Specy`，`Users` → `User`（单数化）
- `isBetweenValues` 不存在 → 用 `isBiggerOrEqualValue` + `isSmallerThanValue`
- `insertReturningId` 不存在 → 用 `customSelect('SELECT last_insert_rowid()')`
- `AlertRecords` 表 drift 2.28.2 无法生成 → 暂时跳过，异常检测用 service 层实时分析
- 扩展方法名冲突：多个 extension 有同名方法 → 全部加 prefix（`getAllSpecies`/`createBird` 等）

## 坑记录
1. CSS `\n` 文字 vs 换行符 — 浏览器无视媒体查询
2. 二进制替换移字节位 → 炸 UTF-8 编码
3. Node.js 正则 `/\n/g` vs `/\\n/g` 匹配差异
4. EJS 必须在文本层面（字符串 replace）操作，不能用二进制字节替换
5. 编写修改 EJS 模板的脚本要注意 `\r\n` vs `\n`
6. `edit` 工具可靠但 Node.js inline `-e` 在 PowerShell 下引号逃逸是灾难
7. JavaScript 语法错误会导致整个 `<script>` 块失效，不仅仅那一行
8. CSS `@media` 缺失闭合 `}` 会让内部规则在错误断点生效
9. Drift 2.x API：insert 用裸值但 write 用 Value<>；数据类名单数化
10. PowerShell 传 JSON 参数是噩梦——单引号/双引号/括号层层嵌套
